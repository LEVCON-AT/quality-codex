# Admin: User-Management

## Permissions {#permissions}

Permission-Matrix in der App:
- Settings → User-Management → Roles

Default-Rollen: owner, admin, editor, viewer, guest.

## Audit-Log {#audit-log}

Privilegierte Aktionen werden geloggt.
Anzeige: Settings → Audit-Log (sichtbar für owner+admin).

## Einladen

1. Settings → Members → "Einladen"
2. E-Mail eingeben + Rolle wählen
3. Eingeladener User erhält Mail mit Akzeptieren-Link

## Rolle ändern

1. Settings → Members → User finden
2. Drop-down "Rolle" → neue Rolle wählen
3. Bestätigen — Audit-Log-Eintrag wird erstellt

## User entfernen

1. Settings → Members → User finden
2. "Entfernen"-Button → Bestätigen
3. User verliert Workspace-Zugriff sofort
4. Audit-Log-Eintrag

## Konto löschen (User-Initiative, DSGVO Art. 17)

User kann das selbst über Profil → "Konto löschen". Detaillierte Konsequenzen werden vor Löschung erklärt.
